# Stage 4: 自动挑刺机制 - 实施完成

## 📋 概述

自动挑刺机制（Phase 2 - Feature 2）已成功实施。该系统让AI自动检测对方输出中的问题，无需用户手动点击"挑刺"按钮。

## ✅ 实施内容

### 1. 数据模型（已在Stage 1完成）
- **Challenge接口扩展**:
  - `challengerAi`: 挑战者（claude/openai/user）
  - `challengeType`: 类型 (factual-error/logical-flaw/omission/unclear/other)
  - `severity`: 严重性 (low/medium/high)
  - `targetSegment`: 被挑刺的具体文本片段
  - `isAutoGenerated`: 是否自动生成
  - `status`: 状态 (pending/accepted/rejected/debated)

- **新增配置类型**:
  - `AutoChallengeConfig`: 自动挑刺配置
  - `AutoChallengeRequest/Response`: API请求/响应

### 2. 自动挑刺逻辑
- **文件**: `src/lib/features/auto-challenge/prompt.ts`
- **核心函数**:
  - `generateAutoChallengePrompt()`: 生成审计prompt
  - `generateStructuredChallengePrompt()`: 生成结构化挑刺prompt

**Prompt策略**:
```
你正在审计另一个AI (Claude/OpenAI) 的输出。

**对方的输出：**
"""
[对方的完整回答]
"""

**审计维度：**
1. 事实错误 — 引用了不存在的事实、数据错误、常识错误
2. 逻辑漏洞 — 推理过程有问题、前后矛盾、因果倒置
3. 重要遗漏 — 遗漏了关键信息、忽略了重要的反例
4. 表述不清 — 概念模糊、术语使用不当、容易引起误解

**输出格式 (JSON):**
{
  "challenges": [
    {
      "type": "factual-error" | "logical-flaw" | "omission" | "unclear" | "other",
      "severity": "low" | "medium" | "high",
      "targetSegment": "被挑刺的具体文本（50字以内）",
      "reason": "挑刺原因（100字以内）",
      "confidence": 0.0-1.0
    }
  ]
}
```

### 3. 挑刺解析器
- **文件**: `src/lib/features/auto-challenge/parser.ts`
- **核心函数**:
  - `parseChallengeResponse()`: 解析AI返回的JSON
  - `deduplicateChallenges()`: 去重相似挑刺
  - `limitChallenges()`: 限制挑刺数量
  - `sortChallenges()`: 按严重性排序
  - `validateChallenge()`: 验证挑刺内容
  - `filterValidChallenges()`: 过滤无效挑刺
  - `calculateChallengeSummary()`: 计算统计摘要

### 4. API Endpoint
- **文件**: `src/app/api/auto-challenge/route.ts`
- **端点**: `POST /api/auto-challenge`
- **流程**:
  1. 接收Claude和OpenAI的消息
  2. 并行调用两个AI进行互相审计
  3. 解析返回的JSON
  4. 去重、验证、限制
  5. 返回挑刺列表和统计

### 5. UI组件

  - 显示目标片段和原因
  - 用户操作：接受/忽略/展开辩论
  - 可折叠/展开详细信息

- **文件**: `src/components/AutoChallengeConfigPanel.tsx`
- **功能**:
  - 启用/禁用自动挑刺
  - 调整置信度阈值（0.5-1.0）
  - 设置每轮最多挑刺次数（1-10）
  - 配置说明和提示

## 📊 技术细节

### 置信度过滤

- **阈值**: 0.7（默认，可调）
- **目的**: 避免误报，只挑刺确信有问题的地方
- **效果**: 提高挑刺质量，减少噪音

### 去重逻辑

- **键**: `${targetMessageId}-${targetSegment}-${challengeType}`
- **目的**: 避免两个AI挑刺同一问题
- **效果**: 减少重复挑刺

### 限制数量

- **默认**: 每轮最多3次
- **范围**: 1-10次可调
- **目的**: 避免过度挑刺，影响体验
- **策略**: 按严重性排序，优先显示重要问题

### 错误处理

- **JSON解析失败**: 记录错误，返回空数组
- **API调用失败**: 记录错误，不影响主流程
- **验证失败**: 过滤掉不符合规范的挑刺

## 🔧 配置选项

### AutoChallengeConfig

```typescript
interface AutoChallengeConfig {
  enabled: boolean;           // 是否启用自动挑刺
  threshold: number;           // 置信度阈值 (0.5-1.0)
  maxChallengesPerRound: number; // 每轮最多挑刺次数
  allowSelfChallenge: boolean;  // 是否允许AI自我挑刺 (暂未实现)
}
```

### 推荐配置

- **宽松模式**: threshold=0.6, max=5
  - 更多挑刺，可能包含误报
  - 适合研究探索

- **平衡模式**: threshold=0.7, max=3 (默认)
  - 平衡准确性和覆盖面
  - 适合日常使用

- **严格模式**: threshold=0.8, max=2
  - 只挑刺高置信度问题
  - 适合生产环境
